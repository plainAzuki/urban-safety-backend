# 研究目的

## 研究背景

都市生活では、鉄道の運行情報、道路交通情報、気象警報、自治体の防災情報などを確認する場面が多い。しかし、これらの情報は複数の公的サイトや交通事業者サイトに分散しており、情報の形式、更新頻度、表現方法も統一されていない。そのため、利用者が短時間で現在の都市安全状態を把握するには、複数サイトを個別に確認し、内容を自分で整理する必要がある。

また、卒業研究の期間中に大規模な災害、鉄道運転見合わせ、高速道路通行止め、避難情報などが必ず発生するとは限らない。日常的な公的情報だけでは、多くの情報源が「通常」または「情報」状態になりやすく、異常時にシステムがどのように表示・検索・要約できるかを十分に確認しにくい。

## 研究目的

本研究では、愛知県および周辺地域の交通・気象・防災などの公的情報を収集し、異なる情報源から得られる都市安全情報を統一的な形式に構造化したうえで、モバイル向けフロントエンドにより可視化・検索可能にする原型システムを設計・実装する。

さらに、日常的な公的情報では異常事例が十分に発生しない問題に対して、研究検証用の模擬イベントデータを導入する。通常時データと模擬異常イベントデータを比較することで、情報表示、自然言語問い合わせ、リスク提示、出典表示の実現可能性を検証する。

## 研究課題

- 複数の公的情報源から得られる都市安全情報を、共通のデータモデルで扱えるか。
- 交通、道路、気象、防災など異なるカテゴリの情報を、同一の画面で分かりやすく表示できるか。
- 日常的な公的情報と模擬異常イベントを明確に区別して提示できるか。
- 自然言語質問に対して、保存済み情報に基づく要約と参照情報を返せるか。
- 通常時と異常時の違いを、卒業研究の評価実験として説明できるか。

## システム概要

本システムは FastAPI バックエンドと React Native / Expo フロントエンドで構成される。バックエンドは公的情報の取得、構造化、SQLite への保存、模擬イベントの投入、自然言語問い合わせへの要約回答を担当する。フロントエンドは都市安全情報の概要、状態別カード、模擬データラベル、詳細情報、問い合わせ結果、参照情報を表示する。

## 本研究の範囲

- 愛知県および周辺地域を対象とした原型システムの設計と実装。
- 公的情報と模擬イベントデータの統一的な構造化。
- 通常時と模擬異常時の比較による可行性検証。
- モバイル画面での都市安全情報の可視化。
- 保存済み情報に基づく自然言語問い合わせと参照情報提示。

## 本研究で扱わないこと

- 実運用可能な防災製品としての完成度保証。
- 災害時の避難判断や移動可否を最終決定する機能。
- すべての交通機関・自治体・道路情報の網羅。
- Verifier Agent による回答正誤判定の有効性検証。
- 模擬データを実際の公的情報として扱うこと。
