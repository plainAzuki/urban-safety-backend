# 実験計画

## 実験目的

本実験の目的は、公的情報と模擬イベントデータを組み合わせた都市安全情報集約・可視化システムが、卒業研究の範囲で可行性を示せるかを確認することである。

## 通常時データによる確認

通常時データでは、公的情報源から取得した情報だけを表示する。多くの場合、鉄道・道路・気象・防災情報は「通常」または「情報」に近い状態になる。この条件では、以下を確認する。

- 情報源名、地域、状態、更新時刻が表示されるか。
- 複数の情報源を同じ形式で扱えるか。
- 自然言語問い合わせに対して、保存済み情報に基づく要約を返せるか。
- 模擬データが含まれない場合に、UI上で不要な警告が出ないか。

## 模擬異常イベントによる確認

模擬異常イベントでは、研究用シナリオをDBに投入し、通常時では発生しにくい異常状態を確認する。対象シナリオは以下である。

- 鉄道運転見合わせ
- 高速道路事故・通行止め
- 大雨・暴風・高潮などの気象警報
- 避難情報
- 空港・駅・主要道路へ向かう際の注意情報
- 複数イベントの同時発生

## 評価観点

| 観点 | 確認内容 |
|---|---|
| 情報源明示 | source と source_url が表示されるか |
| 統一形式 | 鉄道・道路・気象・防災を同一モデルで扱えるか |
| 模擬データ識別 | is_simulated と「模擬データ」表示があるか |
| 状態表示 | 通常、注意、支障、危険などが分かりやすく表示されるか |
| 問い合わせ | 質問に関連する情報と参照情報を返せるか |
| 比較可能性 | 通常時と異常時の違いを説明できるか |
| 限界明示 | 実運用製品ではなく原型システムであることを明示できるか |

## 具体的なテストケース

| ID | 条件 | 操作 | 期待結果 |
|---|---|---|---|
| T01 | 通常時 | `/dashboard?include_simulated=false` を確認 | 公的情報のみが表示される |
| T02 | 通常時 | `/ask` に「今注意すべき情報は？」を送信 | 保存済み情報に基づく要約と参照情報が返る |
| T03 | 模擬異常時 | `multi_event` を投入 | 鉄道・道路・気象・防災の模擬イベントが保存される |
| T04 | 模擬異常時 | フロントで「模擬含む表示」を有効化 | 模擬データラベル付きで異常イベントが表示される |
| T05 | 模擬異常時 | 「中部国際空港へ向かう時に注意すべき情報は？」を質問 | 空港アクセス関連の模擬情報と参照情報が返る |
| T06 | 復帰確認 | 模擬削除を実行 | 模擬イベントが一覧から消える |
