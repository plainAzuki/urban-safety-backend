# サンプル実験結果

## 実験概要

本成果物は、卒業研究の説明に使う通常時・模擬異常時のサンプル結果である。実際の提出時には、実行日時、スクリーンショット、API応答を追加して更新する。

## 通常時シナリオ

### 条件

- `include_simulated=false`
- 公的情報源から取得済みのデータのみを表示

### 期待される表示

- 情報源名、対象地域、状態、更新時刻が表示される。
- 模擬データラベルは表示されない。
- 多くの情報は「通常」「情報」「注意」など、日常的な状態として表示される。

### 自然言語問い合わせ例

質問:

```text
今、愛知県で注意すべき交通・気象情報は？
```

回答例:

```text
保存済みの公的情報に基づくと、愛知県内の交通・気象情報を確認できます。各項目には情報源、対象地域、状態、更新時刻が含まれます。詳細な判断には各公的情報源の最新ページを確認してください。
```

## 模擬異常時シナリオ

### 条件

- `scenario=ollama_random`
- `count=20`
- `dangerous_ratio=0.7`
- `include_simulated=true`
- ローカル Ollama が生成した20件の模擬イベントをDBに投入
- 低リスク・無危険を約3割、高リスク・支障ありを約7割に設定

### 期待される表示

- 「模擬データ」ラベルが表示される。
- 状態として「支障」「運休」「警戒」「危険」が表示される。
- 日常データのみの場合と比べて、リスク表示と参照件数が増える。

### 自然言語問い合わせ例

質問:

```text
中部国際空港へ向かう時に注意すべき情報は？
```

回答例:

```text
質問に関連する保存済み情報として、空港方面の鉄道遅延、空港連絡道路の速度規制、知多半島沿岸の高波・強風注意が確認できます。これらは研究検証用の模擬データです。
```

## 評価表

| 評価観点 | 通常時 | 模擬異常時 | 備考 |
|---|---|---|---|
| 情報源が明示されているか | 確認可能 | 確認可能 | source/source_url を表示 |
| 統一形式で扱えているか | 確認可能 | 確認可能 | category/status/severity を共通利用 |
| 模擬データが識別できるか | 対象外 | 確認可能 | is_simulated とUIラベル |
| 関連情報を返せるか | 確認可能 | 確認可能 | references を返却 |
| 通常時と異常時の違い | 比較可能 | 比較可能 | include_simulated の切替で確認 |
| 卒業研究としての可行性 | 確認可能 | 確認可能 | 原型システムとして評価 |

## 考察

本システムは、分散した公的情報と研究用模擬イベントを統一的な都市安全情報モデルで扱い、モバイル画面で状態、重要度、出典、更新時刻を表示できる。模擬イベントを導入することで、日常時には確認しにくい異常時の画面表示や問い合わせ結果を検証できる。

一方で、本システムは実運用可能な防災製品ではない。公的情報源の仕様変更、取得失敗、LLM要約の不確実性、地域・交通経路の網羅性不足が残る。実際の避難判断や移動判断には、必ず公的機関・交通事業者の最新情報を確認する必要がある。
